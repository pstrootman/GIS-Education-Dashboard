🎯 Timeline Embed System Demo

📋 How to Use the Embed System:

  1. Create & Save Timeline: Sign in to the main app and create a timeline project
  2. Generate Embed Code: In your timeline, go to Timeline Tools → "Generate Embed Code"
  3. Copy & Paste: Copy the generated iFrame code into ArcGIS StoryMaps or any website
  4. Public Access: Anyone can view the embedded timeline without signing in

🔗 Sample Embed URLs

Use these URL patterns for embedding:

https://dynamic-timeline-3f9aa.web.app/embed.html?project=PROJECT_ID https://dynamic-timeline-3f9aa.web.app/embed.html?project=PROJECT_ID&readonly=true&embed=true

📄 Sample iFrame Code for ArcGIS StoryMaps

<iframe src="https://dynamic-timeline-3f9aa.web.app/embed.html?project=YOUR_PROJECT_ID" width="100%" height="600px" frameborder="0" allowfullscreen title="Interactive Timeline"> <p>Your browser does not support iframes. <a href="https://dynamic-timeline-3f9aa.web.app/embed.html?project=YOUR_PROJECT_ID" target="_blank"> View the timeline directly </a>. </p> </iframe>

🔧 System Features

  • ✅ Secure Editing: Only authenticated users can create/edit timelines
  • ✅ Public Sharing: Embedded timelines are viewable without authentication
  • ✅ Firebase Integration: Data is saved to Firebase and accessible publicly for embedding
  • ✅ Responsive Design: Works on desktop, tablet, and mobile devices
  • ✅ ArcGIS StoryMaps Ready: Optimized for StoryMaps embed blocks
  • ✅ Customizable: Control dimensions, read-only mode, and display options

🚀 Test the System

To test the embed system:

  1. Go to the main application
  2. Sign in or create an account
  3. Create a new timeline project with some events
  4. Use "Timeline Tools → Generate Embed Code" to get your embed code
  5. Test the embed URL to see your timeline publicly viewable

🔧 Debug Information

If events aren't showing up in the embed, check the browser console for debug logs. The embed page now includes comprehensive logging:

  • ✅ URL parameter parsing
  • ✅ Firebase initialization status
  • ✅ Database query details and results
  • ✅ Data structure validation
  • ✅ Event processing and filtering
  • ✅ Timeline rendering status

📋 Testing Checklist

1. ✅ Project ID correctly extracted from URL 2. ✅ Firebase initialized and available 3. ✅ Database document exists and accessible 4. ✅ Timeline data structure contains perspectives 5. ✅ Perspectives contain events array 6. ✅ Events have required fields (title, startDate) 7. ✅ Events converted to vis.js timeline format 8. ✅ Timeline visualization updated with events
